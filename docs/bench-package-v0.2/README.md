# Bench Package v0.2

Источник данных:
- `Uroflow_Bench_BOM_Variants_v0.2.xlsx`
- `Uroflow_Bench_SOP_BOM_v0.2.docx`

Пакет предназначен для быстрой сборки воспроизводимого стенда, где есть эталон `Q_ref(t)` по весовому каналу и синхронизация с iPhone (`LED + beep`).

## Файлы
- `bench_overview_v0.2.csv` — сравнение уровней `Low/Mid/High`.
- `bench_bom_flat_v0.2.csv` — полный BOM (включая пояснительные строки).
- `bench_bom_line_items_v0.2.csv` — только технические позиции BOM.
- `bench_bom_summary_v0.2.csv` — свод по обязательным/опциональным позициям.
- `bench_wiring_v0.2.csv` — карта соединений.
- `bench_mechanics_v0.2.csv` — механические параметры стенда.
- `bench_acceptance_criteria_v0.2.csv` — критерии приёмки прогона в датасет.

## Рекомендация по старту
- Для R&D в этом репозитории целевой вариант: `Mid`.
- Причина: управляемый профиль потока, достаточная точность эталона, приемлемая сложность сборки.
- По своду BOM: `Mid` = `12` позиций, из них `10` обязательных.

## Критичные требования из SOP
- Частота логирования массы: не ниже `50 Гц` (для базового стенда допустимо `10 Гц`).
- Ошибка объёма по калибровке: `<= 1%` на `0..1000 мл`.
- Синхронизация с iPhone через `t0` (`LED` в кадре + `beep` в аудио + метка в логе стенда).
- Для включения прогона в датасет: монотонный `m(t)`, валидный интеграл объёма, детектированный `t0`, заполненные метаданные.

## Регенерация из XLSX
```bash
python3 scripts/extract_bench_bom_v0_2.py \
  --xlsx-path "/path/to/Uroflow_Bench_BOM_Variants_v0.2.xlsx" \
  --output-dir docs/bench-package-v0.2
```

## Что заполнить перед закупкой
- Колонки `Ссылка/поставщик (заполнить)` и `Цена (заполнить)` в `bench_bom_line_items_v0.2.csv`.
- Локальные требования по влагозащите и электробезопасности под площадку испытаний.
