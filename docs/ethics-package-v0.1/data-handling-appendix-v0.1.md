# Data Handling Appendix v0.1

Дата: 2026-02-23
Применение: приложение и клинический пилот MVP v0.1

## 1) Принципы

- Data minimization: по умолчанию хранить только производные данные.
- Privacy by default: raw аудио/видео не хранить без отдельного согласия.
- Least privilege: доступ только по роли.
- Traceability: любая запись должна быть трассируема до версии модели.

## 2) Классы данных

Класс A (идентификаторы исследования):
- `study_id`, `site_id`, `subject_id`, `session_id`.

Класс B (медицинские производные данные):
- `Q(t)`, метрики, quality score/reasons, uncertainty.

Класс C (raw media, опционально):
- аудио/видео/глубина (только при отдельном согласии и утвержденной политике).

## 3) Политика хранения

- A/B данные: хранить в защищённом контуре исследования согласно локальной политике.
- C данные: отдельный защищённый контур, ограниченный срок хранения.
- Удаление: по истечении retention или по законному запросу субъекта.

## 4) Контроль доступа

Роли:
- `Clinical Operator` — ввод/просмотр сессий своего центра.
- `Clinical PI` — полный доступ к данным центра.
- `Data Manager` — обезличенные выгрузки и контроль качества.
- `ML Engineer` — только обезличенные наборы для обучения/валидации.
- `RA/QA` — аудитные записи и отчёты.

## 5) Обезличивание и выгрузки

- Перед выгрузкой в аналитику удаляются прямые идентификаторы.
- Ключ сопоставления хранится отдельно у уполномоченного лица центра.
- Для внешнего обмена использовать только de-identified datasets.

## 6) Логи и аудит

Логируется:
- кто и когда создал/изменил/выгрузил запись;
- версия приложения и модели;
- причины quality-отклонений и повторов.

Минимум раз в месяц:
- аудит несанкционированного доступа;
- проверка целостности и полноты логов.

## 7) Incident response

Критический инцидент:
- утечка данных;
- несанкционированный доступ;
- утрата носителя с данными.

Порядок:
1. Немедленная изоляция инцидента.
2. Уведомление ответственных лиц.
3. Оценка масштаба и затронутых субъектов.
4. Корректирующие меры и документирование.

## 8) Региональные оговорки (planning)

- РФ: учитывать требования локализации данных граждан РФ.
- ЕС: health data как special category, legal basis + safeguards.
- Китай: sensitive PI и отдельное согласие.
- США: B2B/HIPAA контур при работе с covered entities/BA.
