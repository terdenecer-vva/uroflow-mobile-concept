# One-Page Clinical Protocol v0.1 (этический комитет / IRB)

Дата: 2026-02-23
Версия: Draft v0.1
Продукт: Smartphone Uroflowmetry App (iPhone)

## 1) Название исследования

Сравнение мультимодального смартфон-урофлоуметра с эталонным клиническим
урофлоуметром при free-flow тестировании.

## 2) Цель

Подтвердить сопоставимость измерений `Qmax`, `Qavg`, `Vvoid`, параметров времени
и формы урофлоуграммы между приложением и эталонной системой.

## 3) Дизайн

Тип: проспективное, одноцентровое, сравнительное, парные измерения.

Схема:
- каждый участник выполняет free-flow тест в стандартный урофлоуметр;
- параллельно записывается iPhone (`audio + video ROI + IMU`, depth если доступен);
- оценка качества записи и классификация `valid/repeat/reject`.

Планируемая выборка: 100-150 пациентов.

## 4) Популяция

Включение:
- взрослые пациенты, которым клинически назначена урофлоуметрия;
- согласие на участие и обработку данных;
- способность выполнить протокол съёмки.

Исключение:
- невозможность получить валидный free-flow тест по клиническим причинам;
- отсутствие согласия на запись/обработку.

## 5) Primary Endpoints

- Согласованность `Qmax` (смартфон vs эталон)
- Согласованность `Qavg`
- Согласованность `Vvoid`

Метрики анализа:
- Bland-Altman (bias, limits of agreement)
- MAE/MAPE
- Корреляция (в дополнение, не как единственный критерий)

## 6) Secondary Endpoints

- Согласованность `Flow time`, `Voiding time`, `TQmax`
- Согласованность класса формы кривой
- Доля записей класса `valid` (приемлемость workflow)
- Повторяемость (subset повторных тестов)

## 7) Процедура визита

1. Получение информированного согласия.
2. Стандартный free-flow тест по клиническому SOP.
3. Одновременная запись смартфоном по фиксированному протоколу съёмки.
4. Фиксация quality-флагов приложения.
5. В клиническом workflow: измерение PVR отдельным методом (вне алгоритма приложения).

## 8) План статистики (MVP-уровень)

- Пре-регистрация primary/secondary endpoint до закрытия базы.
- Анализ per-protocol для `valid` записей + sensitivity-анализ для всех попыток.
- Стратификация минимум по полу и позе.
- Дополнительно: анализ переносимости между разными туалетами/комнатами.

## 9) Data Management

- Идентификаторы: псевдонимизированный `subject_id` и `session_id`.
- По умолчанию хранение деривативов (`Q(t)`, признаки, quality), без raw медиа.
- Raw-медиа только при отдельном согласии и утвержденном retention policy.
- Audit trail: версии модели, параметры запуска, причины отклонения записи.

## 10) Safety / Risk Handling

- Приложение не ставит диагноз и не назначает лечение.
- При низком качестве запись маркируется как недостоверная.
- Клинические решения принимаются по стандартным клиническим данным.

## 11) Минимальный пакет на старт этики

- Этот one-pager протокола.
- Informed consent формы.
- DPIA/privacy-заметка по режиму хранения данных.
- SOP оператора съёмки и чек-лист качества.
- Data Safety & Incident handling краткая инструкция.
